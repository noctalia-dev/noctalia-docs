This page is for settings that affect Noctalia as a whole: text, scale, animations, shadows, the volume popup, keyboard navigation inside panels, and commands that run when shell events happen.

Put these settings in your Noctalia config file, for example `~/.config/noctalia/config.toml`.

```toml
[shell]
ui_scale = 1.0
font_family = "sans-serif"

[shell.animation]
enabled = true
speed = 1.0

[shell.shadow]
blur = 12
offset_x = 2
offset_y = 2
alpha = 0.55
```

Use this page when you want to change the overall feel of the shell. Bar layout, bar widgets, themes, and service settings have their own pages.

Notification daemon ownership is **not** in `[shell]`; use `[notification].enable_daemon` (see [Services](/v5/services/)).

## Shell settings

Global UI settings apply across panels, launcher, control center, and other shell surfaces. This is separate from `scale` on each [Bar](/v5/bar/).

```toml
[shell]
ui_scale             = 1.0
font_family          = "sans-serif"
lang                 = "en"
offline_mode         = false
telemetry_enabled    = false
polkit_agent         = false
password_style       = "default"
avatar_path          = "~/Pictures/avatar.png"
show_location        = true
clipboard_auto_paste = "auto"

[shell.animation]
enabled = true
speed   = 1.0

[shell.shadow]
blur     = 12
offset_x = 2
offset_y = 2
alpha    = 0.55
```

| Key | Meaning |
|-----|---------|
| `ui_scale` | Content scale for control center, launcher, clipboard, and other **non-bar** UI. Independent of `bar.scale` on each bar. Neither value changes Wayland output or HiDPI buffer scale. |
| `font_family` | Primary Pango family for shell text (for example `Inter` or `sans-serif`). Fontconfig handles fallback. |
| `lang` | Override language detection. |
| `offline_mode` | When `true`, blocks **all** outgoing HTTP (weather, community palettes, album art, remote notification icons, and similar). Distros can default this to satisfy strict network policies. |
| `telemetry_enabled` | When `true`, sends one anonymous POST to `api.noctalia.dev/ping` per startup (instance id, version, compositor, OS, RAM, resolutions, UI scale). No personal data. Instance id is a random UUID in `~/.local/state/noctalia/instance.id`. |
| `polkit_agent` | Register Noctalia as the PolicyKit agent on `org.freedesktop.PolicyKit1`. Turn off if another desktop already provides auth prompts. |
| `password_style` | `default` or `random` for polkit and lock input glyph shapes. |
| `show_location` | When `false`, hides weather location text in shell UI. |
| `clipboard_auto_paste` | `off`, `auto`, `ctrl_v`, `ctrl_shift_v`, or `shift_insert`. `auto` uses `Ctrl+V` for image entries and `Ctrl+Shift+V` for text. |

**`[shell.animation]`** - `enabled` turns all animated transitions on or off. `speed` is a multiplier (`1.0` normal, `0.5` half speed, `2.0` double).

**`[shell.shadow]`** - Shared shadow for shell surfaces: `blur` radius (0 disables drawn shadows), `offset_x` / `offset_y`, and `alpha` multiplied by each surface's background opacity. Bars and the dock only set `shadow = true|false`; they do not define their own blur or offset.

## OSD

The on-screen display (for example volume HUD) position:

```toml
[osd]
position = "top_right"
```

Allowed values: `top_right`, `top_left`, `top_center`, `bottom_right`, `bottom_left`, `bottom_center`. Defaults to `top_right`.

## Keybinds

Centralized chords for shell panels (launcher, session, clipboard, wallpaper) and for moving the focus and confirming or canceling actions.

```toml
[keybinds]
validate = ["return", "kp_enter"]
cancel   = ["escape"]
left     = ["left"]
right    = ["right"]
up       = ["up"]
down     = ["down"]
```

Each action is a single chord string or an array of alternative chords.

**Chord form:** plain `key`, or `modifier+key`, or `modifier+modifier+key`.

**Modifiers:** `ctrl`, `shift`, `alt` only.

**`super` is not allowed** - any of `super`, `win`, `windows`, `logo`, `meta`, `mod4` is rejected and fails config parsing.

**Actions:** `validate`, `cancel`, `left`, `right`, `up`, `down`.

## Hooks

Optional shell hooks run commands when specific events occur. Under `[hooks]`, each key is one shell command string or an array of commands run in order. The same `noctalia:` prefix rules apply as in [Idle](/v5/services/#idle) behaviors; see the [IPC overview](/v5/ipc/) for examples.

| Hook | When it runs |
|------|----------------|
| `started` | After Noctalia finishes startup and IPC is ready. |
| `wallpaper_changed` | After a persisted wallpaper path is applied. |
| `colors_changed` | After the theme palette is resolved and terminal templates update. |
| `session_locked` / `session_unlocked` | Compositor session lock state changes. |
| `logging_out` | Before the session panel runs logout. |
| `rebooting` / `shutting_down` | Before reboot or shutdown from the session panel. |
| `wifi_enabled` / `wifi_disabled` | NetworkManager Wi-Fi radio toggles (not on the first snapshot). |
| `bluetooth_enabled` / `bluetooth_disabled` | Default adapter powered state toggles (not on first snapshot). |
| `battery_state_changed` | UPower battery state enum changes. Sets `NOCTALIA_BATTERY_STATE` (for example `Charging`, `Discharging`, `Charged`). |
| `battery_under_threshold` | Charge crosses from above to at or below `battery_low_percent_threshold`. Sets `NOCTALIA_BATTERY_PERCENT`. |

| Field | Type | Default | Purpose |
|-------|------|---------|---------|
| `battery_low_percent_threshold` | int | `0` | 0-100. `0` disables `battery_under_threshold`. For example `15` fires the hook when crossing down to 15% or below. |

```toml
[hooks]
started = "notify-send 'Noctalia' 'Shell started'"
battery_low_percent_threshold = 15
battery_state_changed = "notify-send 'Power' \"Battery: $NOCTALIA_BATTERY_STATE\""
battery_under_threshold = "notify-send 'Power' \"Battery at ${NOCTALIA_BATTERY_PERCENT}%\""
```
