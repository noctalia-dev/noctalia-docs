These commands open, close, or toggle Noctalia UI surfaces.

## General

| Action | Command | Description |
|--------|---------|-------------|
| Print status | <IpcCommand command="status" /> | Print basic shell state as JSON. |
| Reload config | <IpcCommand command="config-reload" /> | Reload config files immediately. |
| Toggle settings | <IpcCommand command="settings-toggle" /> | Open or close the graphical settings window. |

## Bar

| Action | Command | Description |
|--------|---------|-------------|
| Show bar | <IpcCommand command="bar-show" /> [bar-name] [monitor-selector] | Reveal matching bar instance(s). Omit both arguments to show every bar on every output. |
| Hide bar | <IpcCommand command="bar-hide" /> [bar-name] [monitor-selector] | Hide matching bar instance(s) and block edge/pointer reveal until the next show/toggle. Omit both arguments to hide every bar. |
| Toggle bar | <IpcCommand command="bar-toggle" /> [bar-name] [monitor-selector] | Toggle visibility for matching bar instance(s). Omit both arguments to toggle every bar. Does not block edge reveal on hide (use <IpcCommand command="bar-hide" /> to stay hidden until <IpcCommand command="bar-show" />). |
| Set bar auto-hide | <IpcCommand command="bar-auto-hide-set" /> &lt;on\|off&gt; [bar-name] [monitor-selector] | Temporarily enable or disable auto-hide for matching bar instance(s). Omit `bar-name` to update every bar. |

## Scripted widgets

| Action | Command | Description |
|--------|---------|-------------|
| Send event | <IpcCommand command="scripted-widget <widget-name> <target[:bar-name]> <event> [payload]" /> | Dispatch an event to a scripted bar widget. Target can be `focused`, `all`, a monitor selector such as `DP-1`, or a bar-qualified target such as `focused:default`. |

## Panels

| Action | Command | Description |
|--------|---------|-------------|
| Open panel | <IpcCommand command="panel-open <id> [context]" /> | Open a panel by id without toggling it closed when it is already open. Optional `context` works like <IpcCommand command="panel-toggle <id> [context]" />. |
| Close panel | <IpcCommand command="panel-close [id]" /> | Close the active panel, or close the named panel if it is active. |
| Toggle launcher | <IpcCommand command="panel-toggle launcher [query]" /> | Open or close the app launcher. Optional `query` pre-fills the search input (e.g. `noctalia msg panel-toggle launcher "/wall"`). |
| Toggle session menu | <IpcCommand command="panel-toggle session" /> | Open or close the logout, reboot, and shutdown menu. |
| Toggle clipboard | <IpcCommand command="panel-toggle clipboard" /> | Open or close clipboard history. |
| Toggle wallpaper panel | <IpcCommand command="panel-toggle wallpaper" /> | Open or close wallpaper picker. |
| Toggle control center | <IpcCommand command="panel-toggle control-center [tab]" /> | Open or close Control Center. Optional `tab` opens the tab inside control center by default (e.g. `noctalia msg panel-toggle control-center media` )  |

## Wallpaper

| Action | Command | Description |
|--------|---------|-------------|
| Random wallpaper now | <IpcCommand command="wallpaper-random" /> | Switch to a random wallpaper immediately on all monitors. |
| Random wallpaper (one monitor) | <IpcCommand command="wallpaper-random <connector>" /> | Switch to a random wallpaper on the monitor named by **connector** (e.g. `DP-1`). Other monitors are unchanged. Respects per-monitor directory configuration and theme mode. |
| Get wallpaper | <IpcCommand command="wallpaper-get" /> | Print the default wallpaper path persisted in `settings.toml`. |
| Get wallpaper (one monitor) | <IpcCommand command="wallpaper-get <connector>" /> | Print the effective wallpaper path for the monitor named by **connector** (e.g. `DP-1`). |
| Set wallpaper (all monitors) | <IpcCommand command="wallpaper-set <path>" /> | Set every connected monitor and the default wallpaper, persisted to `settings.toml` (per-monitor entries plus `wallpaper.default.path`). Same behavior as choosing **All monitors** in the wallpaper panel. Use an image path (`~` expanded) or `color:#RRGGBB` / `color:#RRGGBBAA`; non-color paths must exist at call time. |
| Set wallpaper (one monitor) | <IpcCommand command="wallpaper-set <connector> <path>" /> | Set only the monitor named by **connector** (e.g. `DP-1` — same value as per-monitor keys in config and the wallpaper panel). Remaining tokens are joined as the path. Other monitors are unchanged. Same path and color rules as the all-monitor form. |

The CLI forwards everything after the command as one string (`noctalia msg wallpaper-set …`), so `noctalia msg wallpaper-set DP-1 /home/you/My Pictures/wall.jpg` still yields a correct path when the shell passes separate arguments after the command (they are rejoined with spaces for parsing).

## Dock

|--------|---------|-------------|

## Notifications

| Action | Command | Description |
|--------|---------|-------------|
| Enable Do Not Disturb | <IpcCommand command="notification-dnd-set on" /> | Hide notification toasts. Notification history still fills. |
| Disable Do Not Disturb | <IpcCommand command="notification-dnd-set off" /> | Show notification toasts again. |
| Toggle Do Not Disturb | <IpcCommand command="notification-dnd-toggle" /> | Toggle notification toast visibility. Notification history still fills. |
| Show Do Not Disturb status | <IpcCommand command="notification-dnd-status" /> | Print the current Do Not Disturb state. |
| Clear active notifications | <IpcCommand command="notification-clear-active" /> | Dismiss all currently visible active notifications. |
| Clear notification history | <IpcCommand command="notification-clear-history" /> | Remove all entries from notification history. |

## Media

| Action | Command | Description |
|--------|---------|-------------|
| Previous track | <IpcCommand command="media previous" /> | Ask the active MPRIS player to go to the previous item. |
| Play/pause toggle | <IpcCommand command="media toggle" /> | Toggle play/pause on the active MPRIS player. |
| Stop playback | <IpcCommand command="media stop" /> | Stop the active MPRIS player and clear it from Noctalia's active-player selection (so bar media widgets can hide when configured to hide without an active player). |
| Next track | <IpcCommand command="media next" /> | Ask the active MPRIS player to go to the next item. |

## Theme

| Action | Command | Description |
|--------|---------|-------------|
| Get theme mode | <IpcCommand command="theme-mode-get" /> | Print the current resolved theme mode: `dark` or `light` (from `[theme].mode` or from the `[location]` schedule when mode is `auto`). |
| Toggle theme mode | <IpcCommand command="theme-mode-toggle" /> | Switch between dark and light theme mode (same as the bar control when `[theme].mode` is not `auto`). |
| Set theme mode | <IpcCommand command="theme-mode-set dark" /> | Persist `[theme].mode` in `settings.toml` and reload. Pass `dark`, `light`, or `auto` (example shows `dark`). |
| Apply templates | <IpcCommand command="templates-apply" /> | Rerender the enabled configured theme templates for the current palette without changing theme mode or firing shell palette hooks. |
| Set wallpaper palette scheme | <IpcCommand command="theme-wallpaper-scheme-set <scheme>" /> | Persist `[theme].wallpaper_scheme` and reload. Used when the palette is generated from the wallpaper image. Recognized values: **Material 3:** `m3-tonal-spot`, `m3-content`, `m3-fruit-salad`, `m3-rainbow`, `m3-monochrome`. **Custom:** `vibrant`, `faithful`, `dysfunctional`, `muted`. |

## Screenshots

Requires compositor support for `wlr-screencopy-unstable-v1`. Uses the global `[shell.screenshot]` output policy (directory, filename pattern, freeze, save to file, clipboard, pipe command). See [Shell settings](/v5/shell/#shell-settings).

| Action | Command | Description |
|--------|---------|-------------|
| Region screenshot | <IpcCommand command="screenshot-region [widget-name]" /> | Start interactive region capture (drag to select; press **`[keybinds].cancel`** to dismiss). |
| Fullscreen screenshot | <IpcCommand command="screenshot-fullscreen" /> | Capture the focused monitor. |
| Fullscreen (pick monitor) | <IpcCommand command="screenshot-fullscreen pick" /> | Click to choose which monitor to capture. |
| Fullscreen (specific output) | <IpcCommand command="screenshot-fullscreen DP-2" /> | Capture a specific connector by name (e.g. `DP-2`, `HDMI-1`). |
| Fullscreen (all monitors) | <IpcCommand command="screenshot-fullscreen all" /> | One combined desktop PNG spanning every monitor. |

## Session

| Action | Command | Description |
|--------|---------|-------------|
| Lock session | <IpcCommand command="session lock" /> | Lock the current session. |
| Suspend system | <IpcCommand command="session suspend" /> | Suspend without locking first. |
| Lock and suspend | <IpcCommand command="session lock-and-suspend" /> | Lock the session, then suspend once the lock is active. |
| Log out | <IpcCommand command="session logout" /> | End the graphical session (compositor-native when available). |
| Reboot | <IpcCommand command="session reboot" /> | Reboot the system. |
| Shut down | <IpcCommand command="session shutdown" /> | Shut down the system. |
