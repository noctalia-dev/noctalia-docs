These commands are useful for media keys, compositor binds, scripts, and custom widgets.

## Volume

| Action | Command | Description |
|--------|---------|-------------|
| Set output volume | <IpcCommand command="volume-set 65" /> | Set output volume. Accepts `65`, `65%`, or normalized values such as `0.65`. |
| Raise output volume | <IpcCommand command="volume-up" /> | Raise output volume by the default step. |
| Raise output by amount | <IpcCommand command="volume-up 10" /> | Raise output volume by a custom step. |
| Lower output volume | <IpcCommand command="volume-down" /> | Lower output volume by the default step. |
| Mute output | <IpcCommand command="volume-mute" /> | Toggle output mute. |

Default step is 5% when not specified. Output volume is clamped to 100% unless `[audio] enable_overdrive = true`, which allows up to 150%.

## Brightness

| Action | Command | Description |
|--------|---------|-------------|
| Set current brightness | <IpcCommand command="brightness-set 65" /> | Set brightness on the current display. |
| Set one display | <IpcCommand command="brightness-set DP-1 0.65" /> | Set brightness for a specific display or monitor selector. |
| Set all displays | <IpcCommand command="brightness-set * 40%" /> | Set brightness on every display. |
| Raise current brightness | <IpcCommand command="brightness-up" /> | Raise brightness by the default step. |
| Raise one display | <IpcCommand command="brightness-up DP-1 10" /> | Raise brightness for one display. |
| Lower all displays | <IpcCommand command="brightness-down * 5%" /> | Lower brightness on every display. |

Targets can be `current`, `all`, `*`, a display id such as `eDP-1` or `DP-1`, or a monitor selector token. Values accept normalized `0.0` to `1.0` or percentage form such as `65`, `65%`, and `5%`.

## Power profile

Requires `org.freedesktop.UPower.PowerProfiles` (same as the Control Center home shortcut). Profile ids come from the compositor/session; common values are `performance`, `balanced`, and `power-saver`.

When the active profile changes (from IPC, the control center, a bar widget, a shortcut, or another app), Noctalia shows a short internal notification: summary **Power mode**, body **Switched to:** the profile name, and an icon for the active mode.

| Action | Command | Description |
|--------|---------|-------------|
| Set power profile | <IpcCommand command="power-set balanced" /> | Activate the named profile. If the service reports a profile list, the name must match one of those entries exactly. |
| Cycle power profile | <IpcCommand command="power-cycle" /> | Switch to the next profile in UPower's ordered list (same order as the bar power widget and control center shortcut). Wraps from the last profile back to the first. Takes no arguments. |
