The dock is an optional app launcher. It can show pinned apps, currently running apps, or both.

It is disabled by default. Turn it on with:

```toml
[dock]
enabled = true
pinned = ["firefox", "code", "kitty"]
```

Use the longer example below when you want to move it, resize icons, change spacing, or enable auto-hide.

```toml
[dock]
enabled             = true
position            = "bottom"   # top | bottom | left | right
active_monitor_only = false

icon_size           = 48
padding             = 8
item_spacing        = 6
background_opacity  = 0.88
background_blur     = true
shadow              = true
radius              = 16
margin_h            = 0
margin_v            = 8

show_running        = true
auto_hide           = false
reserve_space       = false

active_scale        = 1.0
inactive_scale      = 0.85
active_opacity      = 1.0
inactive_opacity    = 0.85
show_instance_count = true

pinned = ["firefox", "code", "kitty"]
```

Shadow **blur, offset, and alpha** come from the global `[shell.shadow]` block. The dock only exposes `shadow = true` or `false`.

## `pinned` matching

Each `pinned` entry is matched to desktop files in this order:

1. Desktop entry ID stem (`"firefox"` matches `firefox.desktop` and `org.mozilla.Firefox.desktop`).
2. `StartupWMClass` in the desktop entry.
3. App `Name` (case-insensitive).
4. Full desktop entry path.

If nothing matches, a **placeholder** slot is kept so dock positions stay stable.

## Active app emphasis

- The focused app uses `active_scale` and `active_opacity`.
- Other apps use `inactive_scale` and `inactive_opacity` (clamped 0.1–1.0 for inactive scale, 0.1–1.75 for active).
- Focus changes **animate** between the two.

## Auto-hide

When `auto_hide = true`:

- The dock does **not** reserve a compositor exclusive zone.
- It **fades out** after the pointer leaves (slow ease-in).
- It **fades in** when the pointer enters a thin edge trigger strip.

Set `reserve_space = true` to keep the exclusive zone even while auto-hidden.
