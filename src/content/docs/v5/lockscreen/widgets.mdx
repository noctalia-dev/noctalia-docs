# Lockscreen widgets

Lockscreen widgets use the same widget types and per-type settings as [desktop widgets](/v5/desktop/widgets/). Layout is stored in a dedicated `[lockscreen_widgets]` section (not shared with `[desktop_widgets]` or `[lockscreen]` backdrop keys).

- Declarative config: `[lockscreen_widgets]` in `~/.config/noctalia/*.toml`
- GUI/editor overrides: `settings.toml` under your Noctalia state directory (same as desktop widgets)

```toml
[lockscreen_widgets]
enabled = true

[lockscreen_widgets.widget.clock_main]
type     = "clock"
output   = "DP-1"
cx         = 1600.0
cy         = 120.0
box_width  = 360.0
box_height = 120.0
rotation   = 0.0

[lockscreen_widgets.widget.clock_main.settings]
format = "{:%H:%M}"
```

When enabled, widgets render on each lock surface, over the same blurred and tinted wallpaper as the rest of the lock screen. To show a clock on the lock screen, add a **clock** widget.

Text widgets (clock, weather, media player, label, sysmon) support a per-widget `font_family` setting just like desktop widgets — empty inherits the shell font. See [Desktop widgets — Common settings](/v5/desktop/widgets/#common-settings).

---

## Edit mode

Edit mode is only available while the session is **unlocked**. The editor shows a dimmed full-screen preview with a movable **login box** preview on each monitor (one per output, move-only).

### Related IPC

For lockscreen widget edit-mode commands, see
[Shell & UI → Lockscreen widgets](/v5/ipc/shell-and-ui/#lockscreen-widgets).

You can also use **Settings → Security → Lock screen → Toggle Editor**.

### Controls

Same interaction model as desktop widget edit mode (move, rotate, resize, grid snap, center guides, toolbar), except the **login box** widget is mandatory per monitor and can only be moved. See [Desktop widgets — Edit mode](/v5/desktop/widgets/#edit-mode).

---

## Config format

The schema mirrors `[desktop_widgets]`: `enabled`, `schema_version`, optional `[lockscreen_widgets.grid]`, `widget_order`, and `[lockscreen_widgets.widget.<id>]` tables with `type`, `output`, `cx`, `cy`, `box_width`, `box_height`, `rotation`, `enabled`, and `[lockscreen_widgets.widget.<id>.settings]`.

Widget IDs generated in the editor use the `lockscreen-widget-` prefix. Login box placement uses fixed IDs `lockscreen-login-box@<output>` with `type = "login_box"` (not shown in the add-widget list).
