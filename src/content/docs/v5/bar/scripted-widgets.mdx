# Scripted Widgets

Custom bar widgets driven by [Luau](https://luau.org) scripts. A script controls a widget's glyph, label, and colors, and can run shell commands, send notifications, and read config settings.

## Configuration

```toml
[widget.my_widget]
type   = "scripted"
script = "~/.config/noctalia/scripts/my_widget.lua"
# any extra keys are readable from Lua via barWidget.getConfig()
my_setting = "value"
```

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `script` | string | `""` | Path to the Luau script file. Absolute paths and `~` paths are used directly; relative paths are resolved from the assets bundle (e.g. `scripts/screen_recorder.lua`). |
| `hot_reload` | bool | `false` | Watch the script file for changes and reload automatically (inotify). Intended for development. |
| `scope` | string | `"instance"` | Runtime scope. `"instance"` creates one script runtime per live widget placement. `"shared"` creates one runtime per widget config key and shares its displayed state across all matching bars. |
| `audio_spectrum` | bool | `false` | Subscribe the script to Noctalia's PipeWire spectrum data and call `onAudioSpectrum`. |
| `audio_spectrum_bands` | number | `16` | Number of spectrum bands passed to `onAudioSpectrum` when `audio_spectrum` is enabled. |

Use `scope = "shared"` for scripts whose `update()` computes global state, such as process status. Shared scripts should not depend on per-bar layout state during `update()`.

## Lua API

### `noctalia.*` — general utilities

| Function | Returns | Description |
|----------|---------|-------------|
| `noctalia.log(msg)` | — | Log a message to shell debug output |
| `noctalia.runAsync(cmd)` | `boolean` | Queue a shell command asynchronously (fire-and-forget). Returns true if the launch was queued. |
| `noctalia.runAsync(cmd, callback, timeoutMs?)` | `boolean` | Execute a shell command off the UI thread and call `callback(result)` when it exits. `result` contains `exitCode`, `stdout`, `stderr`, `timedOut`, `stdoutTruncated`, and `stderrTruncated`. |
| `noctalia.commandExists(name)` | `boolean` | Check if a command exists in PATH |
| `noctalia.processMatches(callback, token, ...)` | `boolean` | Check process command lines on a background worker and call `callback(matches)` with true when one command line contains all tokens. Returns true if the query was queued. |
| `noctalia.copyToClipboard(text, mimeType)` | `boolean` | Queue text for Noctalia clipboard publication with the given MIME type, such as `"text/plain"` or `"text/uri-list"`. Returns true if the copy request was queued. |
| `noctalia.flatpakAppInstalled(appId)` | `boolean` | Check common Flatpak installation paths for an app ID without spawning `flatpak`. |
| `noctalia.portalAvailable()` | `boolean` | Check whether `xdg-desktop-portal` and a supported portal backend are running. |
| `noctalia.focusedOutputName()` | `string?` | Return the preferred interactive output connector name, or nil if none is known. |
| `noctalia.notify(title, body?)` | — | Show an internal notification (normal urgency) |
| `noctalia.notifyError(title, body?)` | — | Show an internal notification (critical urgency) |
| `noctalia.getenv(name)` | `string?` | Read an environment variable. Returns nil if unset. |

Scripted widgets run off the UI thread and do not expose synchronous command execution. Use fire-and-forget `runAsync()` for click-triggered actions, callback `runAsync()` for commands that need stdout/stderr, and callback-based native query helpers for polling. Callback commands default to a 5000 ms timeout, clamp to 60000 ms, and cap stdout/stderr at 1 MiB per stream.

Lua callbacks have execution budgets. If a callback repeatedly exceeds its budget, Noctalia keeps the last good displayed state and stops scheduling that widget until reload.

### `barWidget.*` — widget display

| Function | Returns | Description |
|----------|---------|-------------|
| `barWidget.setText(text)` | — | Set the label text |
| `barWidget.setGlyph(name)` | — | Set the icon by Noctalia alias, Tabler icon name, or `U+` / `0x` codepoint string |
| `barWidget.setFont(familyOrPath)` | — | Set the label font. Pass a system font family name, or a path to a `.otf` / `.ttf` / `.woff2` file (absolute, `~/`, or relative to the assets dir). The font is registered at runtime — no restart needed. |
| `barWidget.setColor(color, mode?)` | — | Set the label color by color role (recommended); fixed hex colors are also supported |
| `barWidget.setGlyphColor(color, mode?)` | — | Set the glyph color by color role (recommended); fixed hex colors are also supported |
| `barWidget.setVisible(visible)` | — | Show or hide the entire widget |
| `barWidget.setUpdateInterval(ms)` | — | Set the `update()` tick interval in milliseconds (default 250, minimum 16) |
| `barWidget.isVertical()` | `boolean` | Whether the bar is in vertical orientation |
| `barWidget.getConfig(key, default?)` | `any` | Read a TOML config setting. Returns the typed value or default/nil. |
| `barWidget.define(manifest)` | — | Declare the widget's name, icon, and settings schema. See [Self-describing widgets](#self-describing-widgets). |

**Color roles:** `primary`, `on_primary`, `secondary`, `on_secondary`, `tertiary`, `on_tertiary`, `error`, `on_error`, `surface`, `on_surface`, `surface_variant`, `on_surface_variant`, `outline`, `shadow`, `hover`, `on_hover`.

Color mode is optional. The default `"auto"` mode lets Settings styling apply to neutral content: unset colors and
`"on_surface"` use the widget foreground resolver, including `color` and `capsule_foreground`. Other roles such as
`"error"` or `"primary"` are treated as script state colors. Pass `"script"` as the second argument to force the script
role even for `"on_surface"`; Settings `color` still has final precedence.

### Lua callbacks

| Callback | Called when |
|----------|------------|
| `update()` | Every ~250ms while the widget is visible |
| `onClick()` | Left mouse button clicked |
| `onRightClick()` | Right mouse button clicked |
| `onMiddleClick()` | Middle mouse button clicked when `[shell].middle_click_opens_widget_settings = false` |
| `onHover(isHovering)` | Mouse enters (`true`) or leaves (`false`) the widget |
| `onIpc(event, payload)` | `noctalia msg scripted-widget ...` dispatches an event to the widget |
| `onAudioSpectrum(values, state)` | Called when `audio_spectrum = true`; both arguments are comma-separated strings. `state` is `audio_active,mpris_playing` as `1` or `0`. |

## Self-describing widgets

A script can declare a **manifest** with `barWidget.define`. The manifest lets Noctalia:

- list the widget directly in the bar's **Add widget** picker (with your label and icon), so it can be added in one click — no manual `[widget.*]` block or `script` path required;
- render your custom settings as native controls in the widget's **Settings** panel.

This works for any script — bundled or your own — with no changes to Noctalia itself.

```lua
-- Must be the FIRST statement in the script (before any side-effecting code,
-- such as starting a process), so Noctalia can read the manifest safely.
barWidget.define({
    label = "Screen Recorder",        -- shown in the picker and settings
    icon = "video",                   -- Noctalia alias / Tabler name / U+codepoint
    description = "Record the screen", -- short picker subtitle
    pickable = true,                  -- optional, default true; false hides it from the picker
    settings = {
        { key = "frame_rate", type = "int", label = "Frame rate",
          default = 60, min = 1, max = 240 },
        { key = "video_codec", type = "select", label = "Video codec", default = "h264",
          options = {
              { value = "h264", label = "H.264" },
              { value = "hevc", label = "HEVC" },
          } },
        { key = "replay_enabled", type = "bool", label = "Replay buffer", default = false },
        { key = "replay_duration", type = "int", label = "Replay seconds", default = 30,
          visible_when = { key = "replay_enabled", values = { "true" } } },
    },
})
```

Each `settings` field accepts:

| Field | Description |
|-------|-------------|
| `key` | Config key the widget reads with `barWidget.getConfig(key)`. Required. |
| `type` | `bool`, `int`, `double`, `string`, `select`, or `color`. |
| `label` | Control label. Defaults to `key`. |
| `description` | Optional helper text under the control. |
| `default` | Default value (matches `type`). |
| `min` / `max` / `step` | Numeric bounds for `int` / `double`. |
| `options` | For `select`: a list of `{ value = ..., label = ... }` (a list of plain strings also works). |
| `advanced` | When `true`, hidden unless "Show advanced" is enabled. |
| `visible_when` | `{ key = "other_key", values = { "true" } }` — only show this field when another setting matches one of `values`. |

Manifest labels are used verbatim and are not translated.

:::note
The manifest call must run before any side effects (for example, before spawning a background process). Noctalia extracts the manifest by running the script up to `barWidget.define` and then stopping, so anything after that line is skipped during extraction.
:::

Bundled scripts that already declare a manifest are discovered automatically from the assets `scripts/` directory; your own scripts get their settings rendered in the GUI once you reference them in a `[widget.*]` block.

## IPC

Scripted widgets can receive named events from Noctalia IPC:

```sh
noctalia msg scripted-widget <widget-name> <target[:bar-name]> <event> [payload]
```

`<widget-name>` is the `[widget.<name>]` config key. `<target>` is required so side-effectful scripts do not accidentally run once per monitor or once per bar. If the target matches more than one live scripted widget placement, the command fails instead of picking one arbitrarily.

| Target | Description |
|--------|-------------|
| `focused` | Dispatch to one matching widget on the preferred interactive output |
| monitor selector | Dispatch to one matching widget on a matching connector, such as `DP-1` |
| `all` | Broadcast to every live matching placement |
| `<target>:<bar-name>` | Narrow any target to a named bar, such as `focused:top` or `DP-1:default` |

The script receives the event through:

```lua
function onIpc(event, payload)
    if event == "toggle" then
        -- ...
    end
end
```

`payload` is the remaining command text after the event, or an empty string when omitted.

## Scene graph

Every scripted widget has the following node tree:

```
InputArea (accepts left / right / middle clicks)
  └─ Flex (horizontal, center-aligned)
      ├─ Glyph (hidden until setGlyph is called)
      └─ Label
```

`setVisible(false)` hides the entire `InputArea` root.

## Example: GPU screen recorder

A `screen_recorder.lua` script is bundled in `assets/scripts/` that wraps `gpu-screen-recorder` with recording and replay buffer support. It declares a [manifest](#self-describing-widgets), so it appears in the **Add widget** picker as "Screen Recorder" and can be added in one click — the TOML below is created for you, and its settings show up in the widget's Settings panel.

- **Left click** — toggle recording
- **Right click** — toggle replay buffer (if enabled) or save replay (if active)
- **Middle click** — save replay buffer
- **IPC** — `start`, `stop`, `toggle`, `replay-start`, `replay-stop`, `replay-toggle`, and `replay-save`

```toml
[widget.screen_recorder]
type             = "scripted"
script           = "scripts/screen_recorder.lua"
scope            = "shared"
directory        = ""
filename_pattern = "recording_%Y%m%d_%H%M%S"
video_source     = "portal"
video_codec      = "h264"
audio_codec      = "opus"
audio_source     = "default_output"
quality          = "very_high"
frame_rate       = 60
color_range      = "limited"
resolution       = "original"
show_cursor      = true
copy_to_clipboard = false
restore_portal   = false
replay_enabled   = false
replay_duration  = 30
replay_storage   = "ram"
hide_inactive    = false
```

```sh
noctalia msg scripted-widget screen_recorder focused start
noctalia msg scripted-widget screen_recorder focused stop
noctalia msg scripted-widget screen_recorder DP-1 toggle
noctalia msg scripted-widget screen_recorder focused:default start
```

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `directory` | string | `""` | Output directory (empty = `~/Videos`) |
| `filename_pattern` | string | `"recording_%Y%m%d_%H%M%S"` | strftime pattern for filenames |
| `video_source` | string | `"portal"` | `portal`, `screen`, or `focused-monitor` (Hyprland) |
| `video_codec` | string | `"h264"` | `h264`, `hevc`, `av1`, `vp8`, `vp9` |
| `audio_codec` | string | `"opus"` | `opus` or `aac` |
| `audio_source` | string | `"default_output"` | `none`, `default_output`, `default_input`, `both` |
| `quality` | string | `"very_high"` | `medium`, `high`, `very_high`, `ultra` |
| `frame_rate` | number | `60` | Recording frame rate |
| `color_range` | string | `"limited"` | `limited` or `full` |
| `resolution` | string | `"original"` | `original`, `1920x1080`, `2560x1440`, etc. |
| `show_cursor` | bool | `true` | Include mouse cursor in recording |
| `copy_to_clipboard` | bool | `false` | Copy file URI to clipboard after recording |
| `restore_portal` | bool | `false` | Restore previous XDG portal session |
| `replay_enabled` | bool | `false` | Enable replay buffer support |
| `replay_duration` | number | `30` | Replay buffer duration in seconds |
| `replay_storage` | string | `"ram"` | `ram` or `disk` |
| `hide_inactive` | bool | `false` | Hide widget when not recording or replaying |

## Example: Bongo Cat

A `bongocat.lua` script is bundled in `assets/scripts/` (with its `bongocat.otf` font in `assets/fonts/`). A cat sits in your bar and slaps its paws when you type or to the beat, naps when idle, and blinks. It declares a [manifest](#self-describing-widgets), so it appears in the **Add widget** picker as "Bongo Cat" and can be added in one click.

- **Left click** — pause / resume
- **IPC** — `keydown`, `keyup`, `keyrepeat` (payload `space` or `enter` for a two-paw slap), plus `pause`, `resume`, `toggle`

```toml
[widget.bongocat]
type           = "scripted"
script         = "scripts/bongocat.lua"
input_device   = "/dev/input/event6"
audio_spectrum = true
tappy_mode     = true
rave_mode      = true
```

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `input_device` | string | `""` | evdev keyboard node to read, e.g. `/dev/input/event6`. Find yours with `sudo evtest` or `ls -l /dev/input/by-id`. Must match `/dev/input/eventN`. |
| `audio_spectrum` | bool | `false` | Enables Noctalia PipeWire spectrum events for sound-reactive modes. |
| `tappy_mode` | bool | `false` | Slap paws to detected beats while audio is active. |
| `rave_mode` | bool | `false` | Flash the cat through the v4 Slow Bongo rainbow colors on detected beats. |
| `use_mpris_filter` | bool | `false` | Only react while the active MPRIS player reports `Playing`. |

Leave `input_device` unset to disable regular keyboard input and use only sound reactivity.

When `input_device` is set and [`evtest`](https://cgit.freedesktop.org/evtest/) is installed, the widget automatically launches **one** background reader that forwards key events back over IPC. The reader is `flock`-guarded, so multi-monitor setups and hot reloads will not spawn duplicates.

**Requirements:** `evtest` must be installed and your user must be in the `input` group (`sudo usermod -aG input $USER`, then re-login).

:::note
The reader keeps running until the device stops emitting or the lock is released. If you change `input_device`, kill the old reader first: `pkill -f 'evtest /dev/input'`.
:::

```sh
noctalia msg scripted-widget bongocat all pause
noctalia msg scripted-widget bongocat all resume
noctalia msg scripted-widget bongocat all toggle
```
